# Prompt Refinement Pop-out — Design

**Date**: 2026-03-21
**Layer**: L2 (Lab)
**Status**: Approved

## Problem

When a user types a prompt in the PromptBar and hits Enter, generation fires immediately with no opportunity to refine intent, select compatible widgets, or provide additional context. This leads to vague generations that miss pipeline integration opportunities.

## Solution

Intercept Enter in the PromptBar to open a centered glass overlay — the **Prompt Refinement Pop-out** — before generation. The pop-out collects additional context, shows compatible pipeline widgets, asks AI-generated clarifying questions, and offers quick toggles. The user confirms with "Generate" to fire the enriched prompt.

## Flow

1. User types prompt, hits Enter
2. PromptBar stores the prompt and opens the pop-out (no generation yet)
3. Pop-out appears as a centered glass modal overlay on the canvas
4. Simultaneously, a lightweight AI call generates 2-3 clarifying questions
5. User refines prompt, answers questions, selects compatible widgets, toggles options
6. User clicks "Generate" — all context is assembled into an enriched prompt and passed to `generator.generate()`
7. Pop-out closes, generation proceeds as normal (streaming preview, etc.)

## Pop-out Sections

### 1. Prompt Refinement Textarea
- Pre-filled with the original prompt
- Larger textarea (3-4 rows) so user can expand their one-liner
- Auto-focused on open

### 2. Compatible Widgets (conditional)
- Only shown when `graphNodes` exist in the pipeline
- Lists widgets with matching I/O ports (derived from `AIGraphContext.orphanedPorts` and `connectionHints`)
- Each item: checkbox + widget name + port names (e.g., "Timer — emits: tick, subscribes: reset")
- Selected widgets are included in the generation prompt as integration targets
- Hidden entirely when the pipeline is empty

### 3. AI Clarifying Questions
- 2-3 questions generated by a lightweight AI call using the existing `AIGenerator`
- System prompt asks for short clarifying questions only (no code generation)
- Each question has a small text input for the answer
- Shows a shimmer/skeleton while loading (~1-2s)
- User can skip any question (leave blank)
- If the AI call fails, this section is silently hidden (non-blocking)

### 4. Quick Toggles
- Row of pill toggles, all off by default:
  - "Interactive" — widget should respond to user input
  - "Dark mode" — widget should support theme tokens
  - "Emit events" — widget should emit events to the pipeline

### Bottom Bar
- "Cancel" (ghost button) — dismisses without generating
- "Generate" (ember glow button) — assembles enriched prompt and fires generation

## Prompt Assembly

The enriched prompt sent to `generator.generate()` is built by concatenating:

```
[Original/refined prompt text]

Additional context:
- [Toggle]: Interactive widget requested
- [Toggle]: Should support dark mode / theme tokens
- [Toggle]: Should emit events to the pipeline
- [Compatible widget]: Wire to "Timer" (emits: tick)
- [Compatible widget]: Wire to "DataView" (subscribes: data.updated)
- [AI Q&A]: Q: "What data format?" A: "JSON array of objects"
- [AI Q&A]: Q: "Should it persist state?" A: "Yes, across sessions"
```

Only non-empty answers and selected items are included.

## Styling

- Glassmorphism: `rgba(20,17,24,0.92)` background, `blur(20px)`, `saturate(1.2)`
- Border: `1px solid rgba(255,255,255,0.08)`
- Border radius: 16px
- Max width: 520px
- Ember accent on Generate button, section labels in `labPalette.textMuted`
- Backdrop: semi-transparent dark scrim behind the overlay
- Entry animation: `sn-drift-up` + `sn-fade-in`
- Escape key or backdrop click dismisses

## Keyboard

- Escape: dismiss (cancel)
- Cmd/Ctrl+Enter: generate (submit)
- Tab: navigate between sections

## Testing

- Pop-out opens on Enter with prompt pre-filled
- Cancel dismisses without generating
- Generate fires with enriched prompt
- Compatible widgets section hidden when pipeline empty
- AI questions show shimmer then populate
- Quick toggles add to enriched prompt
- Escape dismisses
