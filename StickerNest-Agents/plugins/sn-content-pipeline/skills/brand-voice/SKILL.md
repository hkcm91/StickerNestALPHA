## Brand Identity

StickerNest is a spatial operating system — an infinite-canvas platform where widgets, stickers, and pipelines compose into living interactive workspaces. Built by Kimber, a solo self-taught developer, it spans from casual visual organization to full-blown interactive environments with real-time collaboration and VR support.

The brand should feel like the product: creative, capable, and inviting.

---

## Voice Attributes

### 1. Enthusiastic Builder
Kimber is building something she's genuinely passionate about. The voice reflects that — excited about progress, honest about challenges, proud of what's been built. Never fake enthusiasm.

### 2. Technical but Welcoming
Power under the hood, simplicity on the surface. The platform is architecturally sophisticated (7-layer architecture, typed event bus, Yjs CRDTs) but the voice doesn't gatekeep. Technical depth is available for those who want it; everyone else gets the experience.

### 3. Visual-First
StickerNest is a visual platform. The voice should show, not just tell. Prefer screenshots, demos, and visual examples over walls of text. "See this?" beats "Imagine if you could..."

### 4. Community-Oriented
Building with and for creators. The voice invites participation, asks for feedback, celebrates what users build. Not a corporate monologue — a conversation with fellow builders.

## Tone Spectrum

Adjust tone based on context:

```
Casual ◄─────────────────────────────► Professional
(social, community)                    (pitch decks, press)

Technical ◄───────────────────────────► Accessible
(dev docs, architecture)               (tutorials, marketing)
```

| Context | Tone | Example |
|---------|------|---------|
| Twitter thread | Casual + accessible | "Just shipped cross-canvas events. Widgets can now talk to each other across different canvases in real-time." |
| Reddit r/webdev | Casual + technical | "The event bus uses typed pub/sub with a ring buffer for history replay. Emit-to-handler latency target is <1ms." |
| YouTube tutorial | Accessible + visual | "Let me show you how to build your first widget. It's just HTML, CSS, and JS in a single file." |
| Investor pitch | Professional + accessible | "StickerNest is a spatial operating system serving creators, communities, and developers with a 4-tier SaaS model." |
| TikTok | Very casual + visual | "POV: you're building an operating system and the canvas IS the desktop" |

## Messaging Pillars

### "Every canvas is a living system"
The core vision. A canvas isn't static — stickers trigger actions, widgets respond to events, pipelines connect everything. Users aren't arranging objects; they're composing behavior.

### "Your workspace, your way"
Customization and ownership. Infinite canvas, custom widgets, personal themes. The platform adapts to how you work, not the other way around.

### "Build once, run everywhere"
2D canvas, 3D spatial view, VR on Quest 3. Same widgets, same data, same pipelines — different surfaces. Build for one, get all three.

### "From stickers to systems"
The progression from simple to powerful. Start by placing stickers. Add widgets. Wire pipelines. Write scripts. The depth is there when you want it, invisible when you don't.

## Do / Don't

| Do | Don't |
|----|-------|
| "Canvas" — the workspace | "Board", "scene", "stage" |
| "Widget" — interactive programs | "App" or "plugin" (in official content) |
| "Sticker" — visual assets | "Icon" (in technical context) |
| "I built cross-canvas events today" | "We are pleased to announce..." |
| "This was hard — here's what I learned" | "Revolutionary breakthrough" |
| "Try it and tell me what you think" | "Be sure to like and subscribe" |
| "The pipeline connects widget A to widget B" | "Leveraging our ecosystem synergies" |
| Show a screenshot of the feature | Describe the feature in 200 words |
| "Spatial operating system" | "Next-gen platform" |
| Specific metrics ("87% test coverage") | Vague claims ("industry-leading quality") |

## Audience Adaptation

### Casual Users
Focus on the experience. What can they DO? Use simple language, lots of visuals, excitement about possibilities. Avoid technical jargon entirely.

**Example:** "Place stickers on your infinite canvas, add interactive widgets, and watch your workspace come alive."

### Power Users / Creators
Focus on capabilities. What can they BUILD? Show the widget SDK, the pipeline system, the marketplace. Technical depth is welcome but not required.

**Example:** "Build custom widgets with just HTML, CSS, and JS. Publish to the marketplace and let others install your creations."

### Developers
Focus on architecture. How does it WORK? Show the layer system, the event bus, the typed schemas. Full technical depth.

**Example:** "The Widget Bridge Protocol validates every postMessage with Zod schemas. Origin checking is mandatory — no exceptions."

### Investors / Stakeholders
Focus on positioning. What's the MARKET? Show the vision, the traction, the business model. Professional tone, clear metrics.

**Example:** "StickerNest serves three user segments — casual creators, power users, and developers — with a 4-tier SaaS model and creator commerce via Stripe Connect."
